📝 Explicação Passo a Passo
Etapa	O que faz	Por que é importante
1. CSVLoader	
Carrega o CSV e transforma cada linha em um documento de texto.	
Facilita o processamento — cada linha pode ser buscada e analisada separadamente.

2. OllamaEmbeddings	
Gera vetores numéricos para cada documento.	
Permite busca semântica — a IA encontra linhas relevantes mesmo que as palavras sejam diferentes da pergunta.

3. Chroma	
Cria uma base vetorial e persiste em disco.	
Você pode reutilizar a base sem precisar recalcular embeddings toda vez.

4. Retriever	
Define como os documentos relevantes serão buscados (top-k).	
Garante que só informações relevantes sejam passadas ao LLM.

5. Ollama LLM	
Define o modelo de linguagem que vai gerar a resposta.	
Pode ser um modelo pequeno e rápido localmente ou um modelo mais avançado.

6. RetrievalQA	
Junta tudo em um pipeline: busca → contexto → resposta.	
Facilita usar em produção, é só mandar a pergunta e obter resposta pronta.

7. Query	
Pergunta em linguagem natural.	
É a parte que o usuário interage — não precisa escrever SQL.

8. Result	
Mostra resposta + documentos usados.	
Transparência — você sabe de onde veio a informação.
